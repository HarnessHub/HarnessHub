# HarnessHub v0.1.0-rc.1

First public MVP release candidate for the OpenClaw-first HarnessHub image workflow.

## Highlights

- ships the `harness` CLI with `inspect`, `export`, `import`, and `verify`
- publishes the `.harness` portable image format as the current MVP contract
- supports both `template` and `instance` export modes
- validates imported harnesses with manifest-aware readiness checks
- includes a committed OpenClaw end-to-end validation baseline plus a fresh-operator validation path

## Install

```bash
npm install -g harnesshub@0.1.0-rc.1
```

Requires Node.js `>= 20`.

## Acceptance Path

```bash
harness inspect -p /path/to/openclaw
harness export -p /path/to/openclaw -t template -o my-agent.harness
harness import my-agent.harness -t /tmp/harnesshub-import
harness verify -p /tmp/harnesshub-import
```

If `inspect` recommends `instance`, use:

```bash
harness export -p /path/to/openclaw -t instance -o my-agent.harness
```

## Validation

- full release gate run:
  - `npm run build`
  - `npm test`
  - `./scripts/run-agent-preflight.sh`
  - `./scripts/run-cli-smoke.sh`
  - `./scripts/run-openclaw-e2e-validation.sh`
  - `./scripts/run-fresh-operator-validation.sh`
- real local OpenClaw validation record:
  - `docs/validation/openclaw-e2e-validation.md`
- fresh-operator validation record:
  - `docs/validation/fresh-operator-validation.md`

## Current Limits

- OpenClaw is the only production-grade adapter in this RC
- no registry or catalog primitives yet
- no layered parent-image activation yet
- no hosted control plane or SaaS behavior
